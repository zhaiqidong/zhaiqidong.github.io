1. 前言

1.1 名词解释

现场用户
: 指参与活动的观众。

会务人员
: 指组织会务的主办方工作人员。

平台人员
: 指负责运营必肯平台的工作人员

1.3 整体流程/逻辑关系

流程描述：

  1. 平台工作人员在web端后台配置好此次活动的信息。不同的配置类型选项，会影响后续的活动流程。
    1.1 是否需要短信验证手机号
    1.2 是否设置受邀名单
    1.3 活动的配置内容等，如链接地址，卡券核销人员信息等。
  2. 现场用户抵达现场后，通过海报等提示打开蓝牙设备，使用微信摇一摇，摇出活动后，根据配置进入的链接。进入链接，提示用户授权，获取用户昵称和头像。此环节平均用时数分钟。
    2.1 如果设置了受邀名单，则需要用户输入的手机号在受邀名单中，否则无法提交或获取验证码。
    2.2 如果设置了需要接受短信验证，则用户输入完手机号后，会收到短信验证码。验证短信无误，方可提交。
  3. 现场用户入场前活动完成，会收到一张卡券。用户凭借卡券入场。入口处由会务人员扫描二维码，确认用户有效，引导用户快速入场。
  4. 用户完成入场前活动后，现场大屏幕和工作人员的web页面均可实时看到参与信息。
  5. 会务人员在入口处使用微信扫描现场用户的卡券二维码进行卡券核销，核销成功则邀请用户入场。不成功则无法入场。此环节只需要数秒钟。
  6. 活动结束，或活动过程中，会务人员与平台员工均可浏览或导出参会用户信息。

产品逻辑：在活动外围多处放置蓝牙设备和说明海报，引导用户完成较为复杂的会前签到活动。在入口处流程极简，避免阻塞。

通过卡券，引导用户关注公众号，积累用户。

2. 功能

2.1 平台人员活动配置页面（WEB端）

2.1.1 导入邀请名单

2.1.2 设置是否开通短信验证

2.1.3 设置卡券核销人员信息

2.1.4配置页面

2.2 会务人员（B端）

2.2.1 查看会场状况

会务人员可查看多少人到场。

2.2.2 搜索用户

会务人员可根据用户姓名或用户手机号搜索用户。

2.2.3 导出用户信息列表

2.2.4 修改用户信息

防止录入错误，导致受邀嘉宾无法入场。

2.3 用户端（C端）

2.3.1 输入手机号

如果选择了邀请名单，则用户输入手机号时要判断用户输入的号码是否在邀请名单中。

要对用户输入的号码进行有效检验。

2.3.2 接收短信验证码

用户提交手机号后，可点击获取验证码。

点击获取验证码后，倒计时60秒，并将按钮文字设置成“xxs内验证码到达”

2.3.3 验证短信验证码

验证短信无误，则将用户信息保存。

2.4大屏幕

用户活动完成后，从微信接口获取用户头像信息，将头像展示在大屏幕上。
